Provides classes and interfaces for writing UI tests using the WindowTester API.

WindowTester Swing test cases are subclasses of {@link com.windowtester.runtime.swing.UITestCaseSwing} which is a special JUnit test case, tailored for running User Interface (UI) tests. These test cases interact with the UI using an instance of {@link com.windowtester.runtime.IUIContext} which is retrieved from the UITestCaseSwing using it's {@link com.windowtester.runtime.swing.UITestCaseSwing#getUIContext()} method. IUIConext supports common UI actions like entering text, clicking widgets, dragging and dropping components and so on.

UI Tests are made robust using Conditions. Conditions implement the {@link com.windowtester.runtime.condition.ICondition} interface and are commonly used to cause tests to pause until some condition becomes true (for instance once a file is created on the filesystem). Conditions can also be registered with the runtime's {@link com.windowtester.runtime.condition.IConditionMonitor} and used to trigger {@link com.windowtester.runtime.condition.IHandler} instances.
